# 株式会社サンプル ── ナレッジハブ

> 当社の経営方針・業務マニュアル・契約書テンプレート・会議録を集約する社内ナレッジ拠点です。
> AI(Claude Code 等)も、ここを読みに来て社員と同じ前提で応答します。

## 会社概要
- **社名**：株式会社サンプル
- **業種**：不動産業(賃貸仲介・売買仲介・管理)
- **設立**：2010年4月
- **本社**：東京都千代田区
- **従業員数**：32名(2026年5月時点)

## このリポジトリの構成

| フォルダ | 内容 | 関連業務 |
|---|---|---|
| `docs/` | 経営方針・契約書テンプレート・顧客対応マニュアル | マニュアル管理 |
| `meetings/` | 経営会議・営業会議の議事録 | 議事録管理 |
| `proposals/` | 経営判断・施策の提案書 | 提案資料管理 |
| `policies/` | AI への指示書(CLAUDE.md)を含む社内ポリシー | AI ガバナンス |
| `playbooks/` | 業務フロー・チェックリスト | マニュアル管理 |
| `prompts/` | AI(Claude Code)用プロンプト | AIプロンプト管理 |
| `knowledge-base/` | 週次学び・エリア別ノウハウ | ナレッジ共有 |

## 運用ルール

1. **重要な変更は Pull Request で**：経営方針・契約書テンプレートの変更は、関係者レビューを経てから反映
2. **議事録は会議当日中に commit**：当日中にコミットしないと記憶が薄れる
3. **AI への指示は `policies/CLAUDE.md` に集約**：全社員が同じ AI 前提で会話できる
4. **個人情報・顧客情報は記載しない**：本リポジトリは社員教育・社内手順の整備が目的
5. **週次学びは金曜の業務終了前に**：`knowledge-base/weekly-learning-YYYY-MM.md` に追記

## AI(Claude Code)の使い方

社員はそれぞれの AI に質問する形で本リポジトリを参照できます。

例：
- 「うちでは重要事項説明書、最近どう変えてきた？」
- 「先月の営業実績はどうだった？」
- 「新人にエリア感覚を教えるなら、何を伝える？」

AI は `policies/CLAUDE.md` と各業務文書を参照して応答します。
